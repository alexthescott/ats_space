alexthescott.ePaper Clock Dashboard

June 4th, 2024

TLDR: ePaper Raspberry Pi Clock showing weather and Spotify info

Single User + Detailed Weather

Clock with two Spotify User's info

 Project installation instructions and code are hosted in my ePaper-Spotify-Clock repo on GitHub. Written in Python/Bash using Waveshare's 4.2inch ePaper display and ePaper Library, Spotipy for track info, OpenWeathermap and requests for weather info, Pillow to generate/display image, and ImageMagick's command-line tools for image dithering.

Materials:

  • Raspberry Pi (I used the Zero W 2)
  • WaveShare 4.2inch ePaper Display
  • MicroSD Card + the ability to write RaspberryPi OS
  • MicroUSB Cable

    • Motivation:

     Around 2018, I stumbled across and was immediately enamored by Jon Ashcroft's 'Now Playing' Spotify Frame project which displays the album art and track information on a small LCD display connected to a Raspberry Pi. As a music and computer nerd, I knew I wanted to build something similar to permanently sit on my desk.

     There are benefits to using LCD screens for brightness, color, and refresh rate all of which are great fits for his 'web first' approach of building and launching a website in kiosk mode. Although I do eventually structure my ePaper Spotify clock differently, Jon's Now Playing project serves as the most direct inspiration for this project.


    From Jon Ashcroft's Now Playing blog post

    From Brett van Zuiden's UpNext blog post

     In contrast to LCD displays, ePaper displays like those found in Amazon's Kindle are far more power efficient at the cost of low refresh rates, poor color replication, high latency, and greater material cost. In researching similar projects, I found Brett van Zuiden's UpNext project from March of 2020.

     UpNext utilizes a WaveShare 4.2inch ePaper display, Raspberry Pi and Google Calendar API to create, as Brett puts it, a "subtle, unobtrusive digital desktop calendar" which "does one thing and does it well." As a fellow appreciator of single use gadgets, I am radically aligned with Brett's project description and use of ePaper and I knew that my project must follow the same basic design principles and screen choice.
